# MLOPS Lifecycle Structure

This folder groups the project by MLOps lifecycle step so the flow is easy to follow.

## Project Root
- `requirements.txt` installs the notebook, MLflow, Prefect, DVC, and Feast dependencies.
- `FraudClassifier.ipynb` is the beginner-friendly notebook version.

## 01 Data Versioning
- `01_creditcard_dataset.csv.dvc` points to the exact dataset version used for training.
- `01_creditcard_dataset.csv` is the local dataset copy managed by DVC.

## 02 Feature Store
- `feature_repo/feature_store.yaml` configures the local Feast repo.
- `feature_repo/02_fraud_feature_definitions.py` defines the Feast entity, feature view, and feature service.
- `feature_repo/data/02_fraud_feature_source.parquet` is the generated batch feature source.
- `feature_repo/data/02_latest_online_features.json` captures an example online feature lookup.

## 03 Orchestration
- `03_prefect_mlops_pipeline.py` runs the end-to-end flow with Prefect.
- `03_pipeline_params.yaml` stores reproducible pipeline parameters.

## 04 Experiment Tracking
- `04_mlflow.env.example` shows how to configure local or remote MLflow.

## 05 Pipeline Versioning
- `dvc.yaml` defines the reproducible MLOps stage.
- `dvc.lock` stores the exact dependency and output hashes from the latest successful run.

## 06 Model Artifacts
- This folder stores the best model, scaler, metrics, and metadata generated by the pipeline.

## Useful Commands
- `python MLOPS/03_orchestration/03_prefect_mlops_pipeline.py`
- `dvc repro MLOPS/05_pipeline_versioning/dvc.yaml:run_full_mlops_cycle`
- `mlflow ui --backend-store-uri sqlite:///MLOPS/04_experiment_tracking/04_mlflow_tracking.db`
- `feast ui` from `MLOPS/02_feature_store/feature_repo`
